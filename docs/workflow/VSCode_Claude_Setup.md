# VS Code + Claude Code: рабочая схема

## Что реально умеет Claude Code в VS Code

Полезно использовать именно **встроенный терминал VS Code**.
Так Claude получает IDE-функции:
- быстрый запуск из редактора,
- просмотр diff внутри IDE,
- передачу выделенного куска кода,
- шаринг диагностик редактора,
- вставку ссылок на файлы/строки.

## Базовая настройка

1. Открой проект в VS Code.
2. Открой встроенный терминал.
3. Запусти `claude`.
4. Выполни `/config` и проверь:
   - diff viewer включен в auto,
   - IDE интеграция активна.
5. Выполни `/terminal-setup`, чтобы удобнее вставлять переносы строки.

## Как видеть “как Клауд собирает”

### Реалистичный ответ
Прямо как Figma-time-travel или live-canvas Claude тебе не покажет сам по себе.
Но в VS Code ты можешь видеть процесс почти комфортно, если разделишь работу так:

- слева — код и diff,
- справа — запущенный UI preview / emulator / browser preview,
- снизу — Claude Code terminal.

### Практическая схема

#### Окно 1
Claude Code с основным агентом в корне проекта.

#### Окно 2
Эмулятор / Android Studio preview / iOS simulator / web prototype preview.

#### Окно 3
Отдельная ветка или отдельный package для визуального прототипа.

## Лучший рабочий паттерн

### Для инженерии
Поручай Claude:
- создать модуль,
- реализовать контракт,
- написать тесты,
- сделать review.

### Для визуала
Поручай Claude отдельно:
- собрать design tokens,
- сделать 3–5 основных экранов,
- привести стили к единой дизайн-системе,
- сделать dark-first premium aesthetic.
